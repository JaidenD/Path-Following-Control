Implemented structure:

1. Assemble a configuration manifold Q and metric G(q).
   - Constant metrics use analytic manifold Exp/Log.
   - Callable metrics use the numerical geodesic BVP/IVP fallback.

2. Represent the path.
   - Path: piecewise geodesic waypoints.
   - CubicSplinePath: smooth coordinate cubic spline through waypoints, projected back onto Q.
   - ParametricPath: user-supplied gamma(s), with optional gamma_prime(s).
   - Fixed test factory: coordinate_circle_path.

3. Transform q -> (eta, xi).
   - eta is the closest path parameter under ||Log_gamma(eta)(q)||_G.
   - xi is the vector Log_gamma(eta)(q), expected to be metric-normal near a regular closest point.

4. Design decoupled coordinate commands.
   - EtaVelocityController: metric-speed command along the path tangent.
   - EtaPositionController: scalar path-parameter tracking.
   - XiStabilizer: linear transverse stabilization xi -> 0.

5. Map back to robot inputs.
   - ComputedTorquePathFollowingController builds qdot_des and qddot_cmd.
   - RobotModel.inverse_dynamics maps q, qdot, qddot_cmd to u for fully actuated robots.
